# Story 10.1: Predictive Workflow Optimization

Epic: 10 - Strategic Production Orchestration
Status: done

## Summary

Implemented predictive workflow optimization for Marcus using accumulated production telemetry to recommend workflow sequencing, identify likely bottlenecks with preemptive mitigations, and provide explicit accept/modify/override recommendation controls.

## Deliverables

- New predictive optimization engine:
  - skills/bmad-agent-marcus/scripts/predictive_workflow_optimization.py
- New test coverage:
  - skills/bmad-agent-marcus/scripts/tests/test_predictive_workflow_optimization.py
- Marcus capability registry update:
  - skills/bmad-agent-marcus/SKILL.md
- Report output contract:
  - skills/reports/predictive-workflow/*.json (generated by default)

## Acceptance Criteria Trace

- AC1: Suggest optimized workflow sequences based on similar past runs.
  - Satisfied by sequence-frequency analysis over historical completed runs (filtered by course/module/preset and excluding ad-hoc runs).
- AC2: Predicted bottlenecks with preemptive mitigation recommendations.
  - Satisfied by stage duration aggregation and mitigation heuristics for top bottleneck stages.
- AC3: Resource allocation suggestions optimize tool usage and timing.
  - Satisfied by specialist usage frequency summaries and scheduling guidance.
- AC4: User can accept, modify, or override recommendations conversationally.
  - Satisfied by structured options block with accept/modify/override actions in output.

## Validation

- pytest skills/bmad-agent-marcus/scripts/tests/test_predictive_workflow_optimization.py
  - Result: pass
- Assertions validated:
  - Similar-run selection uses matching context and excludes ad-hoc runs
  - Recommended sequence is inferred from historical majority pattern
  - Predicted bottlenecks are ranked by average stage duration
  - Fallback behavior works when DB history is unavailable

## Adversarial Review and Remediation

- Finding: Predictive logic risked leaking ad-hoc run behavior into default recommendations.
  - Remediation: hard exclusion of ad-hoc historical runs and explicit ad_hoc_runs_excluded reporting.
- Finding: Recommendation outputs needed direct conversational control semantics.
  - Remediation: explicit accept/modify/override option payload added to every recommendation result.
