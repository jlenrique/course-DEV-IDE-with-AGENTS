# C1M1 Canvas + CourseArc Allocation Matrix (Exemplar)

**Course**: Intrapreneurship for Physicians, Course 1 Module 1  
**Version**: 1.0  
**Date**: 2026-03-25  
**Status**: Exemplar for replication patterns

## Matrix Overview

This matrix demonstrates optimal platform allocation for a complex module containing multiple instructional patterns: polished guided lesson flow, slide-based presentation, private reflection/playbook work, peer discussion, and both low-stakes and graded assessment checks.

## Platform Allocation Decisions

| Activity/Need | Platform | Reasoning | Integration Pattern | Alternatives Considered |
|---------------|----------|-----------|-------------------|------------------------|
| **Welcome & Hero's Journey framing** | CourseArc | High-presence lesson moment requiring polished sequencing, embedded media, and narrative control | Canvas module launcher → CourseArc lesson | Canvas page (rejected: insufficient narrative control) |
| **Slide-based narrated presentations** | CourseArc delivery + Production stack | Visual slides + narration + multimedia need polished presentation layer | Gamma/Canva production → CourseArc delivery | Canvas + embedded slides (rejected: poor presentation experience) |
| **Embedded podcast/video context** | CourseArc | Freakonomics audio + YouTube videos are part of learning sequence, not separate assignments | Embedded within CourseArc lesson flow | Canvas external links (rejected: breaks narrative flow) |
| **Opening self-reflection discussion** | Canvas Discussion | Required asynchronous discussion for introductions + systemic frustration sharing | Native Canvas discussion board | Qualtrics survey (rejected: social learning requires peer visibility) |
| **Private journal reflection** | OneNote Playbook | "Private course journal" signals persistent learner space across program | Canvas checkpoint for completion attestation | Qualtrics private submission (rejected: not persistent/living) |
| **Chapter knowledge checks (5 questions)** | CourseArc | Low-stakes, embedded-in-lesson, immediate feedback | Canvas gradebook sync for completion tracking | Canvas Quiz (acceptable but less seamless) |
| **Innovation Leader Playbook Entry #1** | OneNote Playbook | Living document across entire program with prestige/continuity | Canvas milestone checkpoints | Qualtrics forms (rejected: too episodic for cumulative work) |
| **SWOT Analysis (Task 1)** | OneNote Playbook | Cumulative, reflective, editable over time | Optional Canvas checkpoint submission | Qualtrics structured survey (secondary role for program analytics) |
| **Idea Portfolio (Task 2)** | OneNote Playbook | Scaffolded recall requiring later reuse throughout course | Canvas checkpoint for completion verification | Canvas assignment (rejected: need persistent editability) |
| **Opportunity Radar (Task 3)** | OneNote Playbook | Foundation for capstone work - must remain editable | Canvas structured checkpoint for top 5 submissions | Qualtrics structured collection (useful secondary for analytics) |
| **M&M Innovation peer evaluation** | Canvas Discussion | Graded, mandatory, requires peer visibility and instructor facilitation | Native Canvas with SpeedGrader integration | CourseArc comments (rejected: insufficient grading infrastructure) |
| **Comprehensive module quiz** | Canvas Quiz | Formal graded assessment requiring gradebook integration | Native Canvas quiz with SpeedGrader | Qualtrics LTI (possible but Canvas cleaner for grading) |

## Strategic Patterns Identified

### CourseArc Optimal Use Cases
- **High-presence lesson moments** requiring narrative control
- **Media-rich content** with embedded videos/audio  
- **Polished presentation** of slide-based content
- **Low-stakes knowledge checks** with immediate feedback

### Canvas Optimal Use Cases
- **Graded assessments** requiring gradebook integration
- **Peer discussions** needing social visibility
- **Course structure** and navigation spine
- **Instructor workflow** (SpeedGrader, analytics)

### OneNote Playbook Optimal Use Cases
- **Private reflection** and journaling
- **Living documents** that evolve throughout program
- **Cumulative artifacts** used in later modules/capstone
- **Persistent learner workspace** beyond single assignments

### Qualtrics LTI Strategic Role
- **Structured data collection** for program analytics
- **Pre/post assessments** for longitudinal measurement
- **Exit tickets** and completion checkpoints
- **Adaptive branching** for complex scenarios

## Integration Choreography

### Canvas → CourseArc Handoffs
- **Pattern**: Canvas module provides structure + launches CourseArc lessons
- **Styling**: Consistent visual theme to prevent "disjointed" feel
- **Navigation**: Clear return paths to Canvas for next activities

### Canvas ↔ OneNote Playbook
- **Pattern**: Canvas checkpoints verify completion without requiring full submission
- **Privacy**: Playbook remains private while Canvas tracks engagement
- **Continuity**: Playbook artifacts referenced in later Canvas activities

### Multi-Platform Analytics
- **Canvas**: Official grades and discussion participation
- **CourseArc**: Lesson completion and interaction patterns  
- **Playbook**: Completion attestations and milestone submissions
- **Qualtrics**: Structured response data and longitudinal metrics

## Quality Assurance Checklist

- [ ] **Transition Smoothness**: No jarring platform switches mid-thought
- [ ] **Data Coherence**: All completion tracking flows to Canvas gradebook  
- [ ] **User Clarity**: Learners understand why they're moving between platforms
- [ ] **Accessibility**: WCAG 2.1 compliance maintained across all platforms
- [ ] **Mobile Optimization**: All platforms function well on physician mobile usage

## Replication Guidelines

### For Similar Modules
This matrix pattern works well for modules with:
- Mixed media requirements (video + slides + discussion)
- Both social and private learning components  
- Cumulative artifacts that span multiple modules
- Professional audience requiring high production value

### Adaptation Signals
Consider different allocation when:
- **Heavy simulation requirements** → More Articulate 360 usage
- **Minimal discussion** → Less Canvas Discussion reliance
- **One-time disposable content** → Less Playbook emphasis  
- **High-stakes assessment focus** → More Canvas Quiz usage

## Version Control Notes

**Next Evolution Triggers**:
- CourseArc API improvements affecting integration patterns
- OneNote replacement (Microsoft changes) requiring new playbook platform
- Qualtrics LTI feature updates enabling new interaction types
- Canvas interface changes affecting user experience

**Measurement Success Criteria**:
- Learner completion rates >90% across all platforms
- Transition confusion <10% in post-module feedback
- Instructor workflow efficiency (SpeedGrader usage, discussion management)
- Platform-specific engagement metrics meeting benchmarks

This exemplar should be referenced for similar multi-platform medical education modules requiring sophisticated instructional design allocation.
