# PPTX Import Workflow (Canva Manual Tool Path)

This workflow standardizes slide conversion into Canva while preserving fidelity.

## Preconditions

1. Source deck conforms to docs/app-design-principles.md and style-bible constraints.
2. If source deck is generated by Gamma, verify output assumptions from skills/bmad-agent-gamma/SKILL.md (font substitution, asset embedding, and layer order risks).
3. Asset links and attribution metadata are finalized.
4. Slide notes include narration or facilitator guidance if required.

## Steps

1. Export or save source as `.pptx`.
2. In Canva, select import presentation and upload the `.pptx` file.
3. Validate each slide for font fallback, text overflow, and alignment drift.
4. Normalize brand colors and heading styles using approved palette rules.
5. Rebuild unsupported elements (complex charts, embedded media) manually.
6. Re-run accessibility checks (contrast, minimum font size, reading order).
7. Export deliverables in required formats (`pptx`, `pdf`, optional image assets).

## Common Drift Patterns

- Font substitution changes line wrapping and card heights.
- Grouped vector assets shift z-order during import.
- Bullet indentation can collapse when source uses custom tabs.

## Mitigations

1. Apply template primitives before fine-grained edits.
2. Lock key layout anchors (title, footer, logo) first.
3. Use a side-by-side visual diff pass against original slides.
4. Capture unresolved drift in implementation artifacts with rationale.
