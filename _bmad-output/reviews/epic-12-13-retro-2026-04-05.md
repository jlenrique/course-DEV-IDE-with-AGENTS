## What Went Well

### 1. Party Mode consensus-driven design (both epics)
Every non-trivial design decision was routed through Party Mode for team consensus before implementation. This prevented rework and caught architectural misalignments early. Examples:
- Epic 12: Entire epic scope and story breakdown designed in one Party Mode session
- Story 13.1: Script placement decision (Option A) resolved unanimously before writing code
- Story 13.2: Parameter location (`narration-script-parameters.yaml` vs `run-constants.yaml`) settled before implementation

### 2. Clean lane separation (Epic 13)
The decision to place perception enforcement under Irene's skill directory (not extend Marcus's validator or sensory bridges) preserved the lane matrix boundaries. Two scripts at two timing points: pre-narration (Irene-owned) and post-Pass-2 (Marcus-owned). No coupling, no regression risk.

### 3. Reuse-first approach (both epics)
Epic 12 extended existing `gamma_operations.py` rather than creating new dispatch infrastructure. Epic 13 reused the sensory bridges perception infrastructure, bridge_utils `perceive()` dispatcher, and existing perception schema — no duplication.

### 4. Test-first confidence (both epics)
Every story shipped with comprehensive tests covering all acceptance criteria. The `perceive_fn` injection pattern in Story 13.1 made testing deterministic without mocking internal bridge implementation. Total test count grew from 192 to 357 with zero regressions.

### 5. Prompt pack co-evolution (Epic 12)
Prompt pack v4.1 was updated simultaneously with the code infrastructure, preventing divergence between the production workflow documentation and the actual implementation.

## Challenges

### 1. Hyphenated directory import complexity
Python skill directories use hyphens (`bmad-agent-content-creator`) but Python imports require underscores. Required conftest.py with explicit `importlib.util` loading and `sys.modules` stub registration. Pattern now established but adds friction for every new skill script directory.

**Root cause:** Repository naming convention optimized for readability, not Python imports.
**Mitigation applied:** Created reusable conftest pattern; `skill_module_loader.py` provides shared utilities.

### 2. conftest.py collision across test directories
Running `pytest` across multiple skill test directories simultaneously causes conftest collisions. Must run skill tests in separate pytest invocations.

**Root cause:** Each skill's conftest registers its own `sys.modules` stubs, which conflict when loaded together.
**Workaround:** Run test suites in separate invocations (documented in `next-session-start-here.md` gotchas).

### 3. E2E double-dispatch validation still pending (Epic 12)
Epic 12 code + prompt pack are complete, but no production run with `DOUBLE_DISPATCH: true` has been executed. This was flagged in `next-session-start-here.md` as the first action for the next session but was deferred.

**Risk:** The full pipeline integration has not been validated end-to-end.
**Action:** Schedule E2E test before starting Epic 14.

## Key Insights

### 1. Agent skill directories are the right code boundary
Story 13.1's script placement decision confirmed that agent skill directories (`skills/bmad-agent-{name}/scripts/`) are the correct ownership boundary for agent-specific behavioral logic. Shared infrastructure (bridges, utilities) stays in shared skills. Agent-specific contracts stay with the agent.

### 2. Narration parameters belong in style config, not run constants
`run-constants.yaml` is for pipeline-level configuration (run identity, dispatch mode, quality preset). Narration style parameters (visual references per slide, tolerance, deictic style) belong in `narration-script-parameters.yaml`. This distinction will be useful for Epic 14 motion parameters.

### 3. Downstream reference updates are lightweight but high-value
Story 13.3 showed that extending downstream agents (Vera G4, Quinn-R, Compositor) with new validation criteria is mostly reference documentation updates — low implementation cost but high fidelity assurance value. The G4-08 visual reference traceability check and Quinn-R VR capability were single-line additions with outsized downstream benefit.
