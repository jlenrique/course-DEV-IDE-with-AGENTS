# Branching Scenario Design Guide

1. Define decision points tied to objective mastery.
2. Keep branch fan-out manageable (prefer 2 to 3 paths per node).
3. Ensure every branch has meaningful feedback and recovery path.
4. Tag each branch with competency signal (safe/unsafe/optimal).
5. Include remediation loop for incorrect high-risk decisions.
6. End states must map to assessment or reflection prompts.
